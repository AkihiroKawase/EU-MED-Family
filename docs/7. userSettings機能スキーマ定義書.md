## 7.1. userSettings コレクションの運用ルール
- userSettingsコレクションは、各ユーザー固有の設定情報（通知のON/OFFなど）を保持します。
- ドキュメントIDには、対象となるユーザーのID（userId）をそのまま使用します。 これにより、1ユーザーに対して1つの設定ドキュメントが紐づきます。
- ユーザーがアカウントを作成した（または初めて設定を変更した）タイミングで、このドキュメントが作成されます。
- アプリの設定画面から、ユーザーは自身のuserSettingsドキュメントのみを更新できます。

## 7.2. フィールド: userSettings に必要なフィールド
- enableNotifications: (Boolean) プッシュ通知の全体的な許可設定。デフォルトは true。
- updatedAt: (Timestamp) この設定が最後に更新された日時。
- （※将来的に「いいねの通知のみOFF」など、より詳細な設定項目が必要になった場合は、notificationPreferences: { likes: boolean, comments: boolean } のようなMap（オブジェクト）型フィールドとして拡張することを検討します。）

## 7.3. インデックス（推奨）
- このコレクションはドキュメントID（＝userId）で直接参照（GET）することが唯一の用途のため、追加のインデックスは不要です。
