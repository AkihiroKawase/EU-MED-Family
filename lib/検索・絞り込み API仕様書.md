# 検索・絞り込み API仕様

## 1. エンドポイント

`GET /posts`

## 2. クエリパラメータ

| パラメータ | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `q` | String | 任意 | **キーワード検索。** 投稿のメタデータ（Title, Summary）を対象とする。 |
| `category` | String | 任意 | **カテゴリ絞り込み。** 単一のカテゴリ名（完全一致）を指定する。 |
| `tags` | String | 任意 | **タグ絞り込み。** タグ名をカンマ区切りで複数指定可能（例: `tag1,tag2`）。 |

## 3. 検索ロジック

### 3.1. 全体の整合条件

* `q`, `category`, `tags` の各条件が指定された場合、すべて **AND条件** で結合されます。
* （例: `q` に一致し、 **かつ** `category` に一致し、 **かつ** `tags` の条件に一致する投稿）

### 3.2. タグ（`tags`）の内部ロジック

* `tags` パラメータに複数のタグが指定された場合（例: `tags=ハンガリー,生活`）、それらのタグは **OR条件** で評価されます。
* （例: 「ハンガリー」タグを持っている、 **または** 「生活」タグを持っている投稿）

## 4. 具体例

### 例1: キーワードのみ

`GET /posts?q=試験`

* メタデータに「試験」を含むすべての投稿を取得します。

### 例2: カテゴリとタグ（OR）

`GET /posts?category=臨床実習&tags=ハンガリー,生活`

* カテゴリが「臨床実習」であり、
* **かつ**
* タグに「ハンガリー」 **または** 「生活」のいずれか（もしくは両方）が含まれる投稿を取得します。
